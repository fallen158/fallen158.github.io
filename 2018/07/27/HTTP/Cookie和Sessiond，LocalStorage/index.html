Cookie

什么是Cookie？

  1. Cookie 是浏览器访问服务器后，服务器传给浏览器的一段数据。浏览器需要保存这段数据，不得轻易删除。
  2. 浏览器需要保存这段数据，不得轻易删除。
  3. 此后每次浏览器访问该服务器，都必须带上这段数据。
  4. Cookie 默认在用户关闭页面后就失效，后代代码可以任意设置Cookie 的过期时间

Cookie的作用

Cookie 主要用来分辨两个请求是否来自同一个浏览器，以及用来保存一些状态信息。它的常用场合有以下一些。

  • 会话状态管理（如用户登录状态、购物车、游戏分数或其它需要记录的信息）
  • 个性化设置（如用户自定义设置、主题等）
  • 浏览器行为跟踪（如跟踪分析用户行为等

以前的开发者使用 Cookie 作为客户端储存。这样做虽然可行，但是并不推荐，因为 Cookie 的设计目标并不是这个，它的容量很小（4KB），缺乏数据操作接口，而且会影响性能。客户端储存应该使用 Web LocalsStorage API

Cookie的常用API

  • expires： 到期时间，具体默认根据浏览器的定义。
  • domain ： cookie生效的域名，默认当前。
  • path ： cookie生效的路径，默认当前。
  • secure ： 是否只在https下生效，默认可以。
  • httponly 是否允许JS获取，默认可以。

Session

什么是Session

  1. 将SessionID（随机数）通过Cookie发给客户端
  2. 客户端访问服务器时，服务器读取SessionID
  3. 服务器有一块内存（哈希表）保存了所有session
  4. 通过SessionID我们可以得到对应用户的隐私信息，如 Id，email
  5. 这块内存（哈希表）就是服务器上的所有session

这样用户无法修改。而且sessionId不存在内存里，如果不小心删除，重新登录即可。

localStorage

什么是localStorage

  1. localStorage 跟 HTTP 无关
  2. HTTP 不会带上 localStorage 的值
  3. 只有相同域名的页面才能互相读取 localStorage （没有同源那么严格）
  4. 每个域名localStorage 最大存储值为 10MB左右 （每个浏览器不一样）
  5. 常用场景：记录有没有提示过用户（没有用的信息，不能记录密码）
  6. localStorage 永久有效，除非用户清理缓存

localStorage常用的API

  • localStorage.setItem：接收两个参数，key和value
  • localStorage.getItem：通过key，获取值
  • localStorage.clear：请空所有localStorage

localStorage应用场景

例如，一个网站更新改版啦，需要提示每一个进来的用户。

这么做的好处， 如果网站改版已经提示啦，就会存储一个已经提示了的消息，在localStorage里，下次用户再进来就不会一直提示啦。

SessionStorage

1，2，3，4和localStorage一样

6.SessionStorage 在用户关闭页面（会话结束后）就失效

CooKie和Session的关系

  1. Cookie是Session的基石
  2. 一般来说，Session 基于 Cookie 来实现。
  3. Cookie 保存在客户端，每次都随请求发送给 Server
  4. .Session 保存在 Server 的内存里，其 SessionID 是通过 Cookie 发送给客户端的

Cookie和LocalStorage的区别

  1. Cookie 在每次请求会被浏览器带到服务器（会加1kb）
  2. Cookie 最大存储为4KB左右
  3. LocalStorage跟HTTP无关
  4. LocalStorage 最大存储为10MB左右
