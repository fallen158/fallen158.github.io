什么是路由

传统的路由指的是：当用户访问一个URL时，对应的服务器会接收这个请求，然后解析url中的路径，从而执行对应的处理逻辑。这样就完成啦一次路由并发。例如：

简单距举例说明：假如我们有一台提供Web服务的地址是：http:/127.0.0.1，而该Web服务又提供了三个可供用户访问页面，其URL是：

  • http:/127.0.0.1/
  • http:/127.0.0.1/tab1
  • http:/127.0.0.1/tab2

那么其路径就分别是 /，/tab1，/tab2。

当用户使用 http:/127.0.0.1/tab1 来访问该页面时，Web 服务会接收到这个请求，然后会解析 URL 中的路径 /tab1，在 Web 服务的程序中，该路径对应着相应的处理逻辑，程序会把请求交给路径所对应的处理逻辑，这样就完成了一次「路由分发」，这个分发就是通过「路由」来完成的。

以前路由都是后台做的，通过用户请求的url导航到具体的html页面，前端路由就是通过配置js文件，把这个工作拿到前端来做。

简单的说，就是你给我一个路径，我给你返回一个页面（后端路由）

你给我一个路径，我给你返回一个状态（前端路由）

前端路由

前端的路由和后端的路由在实现技术上不一样，但是原理都是一样的。在 HTML5 的 history API 出现之前，前端的路由都是通过 hash 来实现的。

前端路由主要做的就是两点：

  • 在页面不刷新的情况下实现url的变化
  • 捕捉url的变化，根据url更改页面内容

例如：我们做一个状态切换，怎样能让我们的页面状态在刷新以后不改变呢

  • 使用hash保存当前页面状态信息，使用。缺点：容易受到其他a标签影响
  • 用HTML5 history API 使用路径代替锚点表示状态，例如
  • 使用路径的话肯定会跳转页面，但我们可以使用事件阻止其默认事件，使其不能跳转页面，而且不会受到其他a标签锚点的影响。

hash

hash是url中#后面的部分，服务器会自动忽略这部分，但是前端可以通过location.hash来获取。

这是一个tab点击高亮事件，通过hash#，记录值，分享给其他用户，就可以得到同样的页面

Web 服务并不会解析 hash，也就是说 # 后的内容 Web 服务都会自动忽略，但是 JavaScript 是可以通过 window.location.hash 读取到的，读取到路径加以解析之后就可以响应不同路径的逻辑处理。
