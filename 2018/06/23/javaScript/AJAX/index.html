AJAX和JSON

发表于 2018-06-23 | 分类于 javaScript

为什么需要AJAX？

回顾下以前是如何发送请求的：

  • 用 form 可以发请求，但是会刷新页面或新开页面
  • 用 a 可以发 get 请求，但是也会刷新页面或新开页面
  • 用 img 可以发 get 请求，但是只能以图片的形式展示
  • 用 link 可以发 get 请求，但是只能以 CSS、favicon 的形式展示
  • 用 script 可以发 get 请求，但是只能以脚本的形式运行

看上面请求方式，都有很明显的缺点，那么有没有什么方式可以实现：

  1. get、post、put、delete 请求都行
  2. 想以什么形式展示就以什么形式展示

就是AJAX。

AJAX的历史

Jesse James Garrett 讲如下技术取名叫做 AJAX：异步的 JavaScript 和 XML。

AJAX的核心是XMLHttpRequest (XHR) API。XHR 是一种用于在 Web 浏览器和 Web 服务器间传输数据消息的 JavaScript API。它支持浏览器使用 HTTP POST（将数据传到服务器）或 GET 请求（从后台服务器访问数据）。

在 2000 年，XHR 最先出现于 IE 5 中。最初是由 Alex Hopmann 编写的 Microsoft ® ActiveX® 控件，创建 XHR 是为了处理 Microsoft Outlook® Web Access，旨在解决高级（当时）前端接口和 Microsoft Exchange Server 间的交互。

IE 5 率先在 JS 中引入 ActiveX 对象（API），使得 JS 可以直接发起 HTTP 请求。随后 Mozilla、 Safari、 Opera 也跟进（抄袭）了，取名 XMLHttpRequest，并被纳入 W3C 规范。

真正的变化是几年后谷歌的Gmail服务，这一项Ajax 技术的使用使得 Web 开发界沸腾起来。它的响应能力和交互性对公众而言是全新的。

readState对象有5种状态

  • 0: UNSENT。表示对象已创建
  • 1: OPENED。表示 open() 方法已成功调用
  • 2: HEADERS_RECEIVED。表示来send()方法已经被调用, 响应头和响应状态已经返回.
  • 3: LOADING。表示响应体正在下载中，responseText中已经获取了部分数据.
  • 4: DONE 。整个请求下载过程已经完毕.

AJAX可以任意设置请求header部分

  • open() 发起服务器连接。可以带参数：
    1. method。将要使用的 HTTP 方法（可以是 POST 或者 GET）
    2. url。请求的 URL
    3. async。 可选布尔参数，表明请求是否异步（该参数的默认值为 Ture）
    4. user。一个可选用户名，供认证使用
    5. password。一个可选密码，供认证使用
  • setRequestHeader() 设置请求头，带有两个参数：报头及其相关值
  • send() 发送请求。该方法带有一个可选参数，包含 POST 请求的正文

AJAX可以获取任意响应的header部分

  • status。请求的标准 HTTP 状态（例如，成功请求将返回 200）
  • statusText。包括 Web 服务器返回的完整响应字符串，其中包括响应文本（例如，304 Not Modified）
  • getResponseHeader()。返回响应头内容；请求报头名称是其惟一参数
  • getAllResponseHeaders()。返回所有响应头的文本
  • responseXML。响应文本的 XML 表示，一个包含 DOM 和所有相关 DOM 方法的文档片段
  • responseText。响应文本的字符串表示
